69 Those are the kind of platforms that are fun... so a bunch of

72 -----------------
73 
74 it is also the case that not all addresses will be used if all "cells" are
75 of the same size, 8 bytes...
76 
77 value 0: 0000 0000
78 value 1: 0000 0008
79 value 3: 0000 0010
80 value 4: 0000 0018
81 
82 Means bits 0,1,2 will always be empty in a valid address.
83 
84 Cons cells also need to be have room for 2 pointers. So each allocated cell from
85 memory should be 8bytes.
86 
87 Things that needs to be represented within these bits:
88 
89  - GC MARK one per cell
90  - TYPE: type of CAR and type of cons
91 
92 Types I would want:
93  - Full 32bit integer. Does not leave room for identification of type
94  - Float values. Same problem
95 
96 
97 Free bits in pointers 64MB heap:
98 31 30 29 28 27 26 2 1 0
99 0 0 0 0 0 0 XX XXXX XXXX XXXX XXXX XXXX X 0 0 0
100 
101 
102 Information needed for each cell:
103  Meaning | bits total | bits per car | bits per cdr
104  GC mark | 2 | 1 | 1 - only one of them will be used (the other is wasted)
105  Type | 2x | x | x
106  Ptr/!ptr | 2 | 1 | 1
107 
108 
109 Types (unboxed):
110  - Symbols
111  - 28bit integer ( will need signed shift right functionality )
112  - 28bit unsigned integer
113  - Character
114 
115 If four types is all that should be possible (unboxed). then 2 bits are needed to differentiate.
116 2 + 1 + 1 = 4 => 28bits for data.
117 
118 bit 0: ptr/!ptr
119 bit 1: gc
120 bit 2-3: type (if not ptr)
121 bit 3 - 24 ptr (if ptr)
122 bit 4 - 31 value (if value)
123 
124 An unboxed value can occupy a car or cdr field in a cons cell.
125 
126 types (boxed) extra information in pointer to cell can contain information
127  - 32 bit integer
128  - 32 bit unsigned integer
129  - 32 bit float
130 
131 boxed representation:
132  [ptr| cdr]
133  |
134  [Value | Aux + GC_MARK]
135 
136 Kinds of pointers:
137  - Pointer to cons cell.
138  - Pointer to unboxed value (fixnums not in a list, I hope this is so rare that it can be removed )
139  - integer
140  - unsigned integer
141  - symbol
142  - float
143  - Pointer to boxed value.
144  - 32 bit integer
145  - 32 bit unsigned integer
146  - 32 bit float
147  - (Maybe something else ? Vectors/strings allocated in memory not occupied by heap?)
148  - vector of int
149  - vector of uint
150  - vector of float
151  - vector of double
152  - String
153

156 is also possible
